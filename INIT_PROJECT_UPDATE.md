# 项目初始化流程更新说明

## 更新概述

本次更新在 `init_project.py` 中加入了数据库初始化流程，并将本地生成的RSA密钥自动设置为初始管理员的密钥。

## 主要变更

### 1. `init_database.py` 更新

#### 新增用户表创建
- 在 `get_create_table_sql()` 函数中添加了 `users` 表的创建SQL
- 包含完整的用户字段：用户名、密码、密钥对、权限等

#### 新增管理员用户创建函数
- `create_admin_user()`: 创建初始管理员用户
- 支持设置用户密钥对
- 如果用户已存在，可以更新密钥

#### 更新验证和导出函数
- 在验证函数中检查用户表
- 在导出的SQL中包含默认管理员用户

### 2. `init_project.py` 更新

#### 新增数据库初始化步骤
- 在第8步添加了 `init_database_and_admin()` 方法
- 自动调用 `init_database.py` 中的相关函数
- 读取本地生成的密钥文件并设置给管理员用户

#### 更新步骤编号
- 将原来的7步流程扩展为8步
- 所有步骤编号相应调整

#### 更新文档说明
- 在模块文档字符串中加入第8步说明
- 在总结中加入数据库初始化信息

## 新的初始化流程

1. **[1/8] 检查前置条件** - 验证Python版本等
2. **[2/8] 生成 RSA 密钥对** - 创建加密密钥文件
3. **[3/8] 配置数据库连接** - 设置数据库连接信息
4. **[4/8] 配置 API 服务** - 设置API服务参数
5. **[5/8] 配置跟单模式** - 设置跟单比例
6. **[6/8] 配置 Redis 缓存** - 可选的缓存配置
7. **[7/8] 生成配置文件** - 创建config.py等配置文件
8. **[8/8] 初始化数据库和管理员账户** - 🆕 **新增步骤**
   - 创建数据库（如果不存在）
   - 创建所有必需的数据表
   - 初始化默认数据
   - 创建管理员用户（用户名：admin，密码：admin123）
   - 将本地生成的RSA密钥设置为管理员的密钥对
   - 验证数据库结构
   - 导出数据库结构文件

## 管理员账户信息

### 默认管理员
- **用户名**: `admin`
- **密码**: `admin123`
- **权限**: 超级管理员
- **密钥**: 自动从本地生成的RSA密钥对设置

### 密钥设置
- 私钥：从 `quant_id_rsa_pkcs8.pem` 读取
- 公钥：从 `quant_id_rsa_public.pem` 读取
- 自动存储到数据库用户表的 `private_key` 和 `public_key` 字段

## 数据库表结构

### users 表字段
```sql
- id: 主键
- username: 用户名（唯一）
- password_hash: 密码哈希值
- private_key: 用户私钥（PEM格式）
- public_key: 用户公钥（PEM格式）
- is_superuser: 是否为超级管理员
- is_active: 账户是否激活
- created_time: 创建时间
- updated_time: 更新时间
- last_login_time: 最后登录时间
- last_activity_time: 最后活跃时间
- login_count: 登录次数
```

## 优势

1. **一站式初始化**: 运行一次 `init_project.py` 即可完成所有设置
2. **自动密钥管理**: 生成的密钥自动设置给管理员，无需手动操作
3. **完整的用户系统**: 建立完整的用户认证体系
4. **即开即用**: 初始化完成后可直接使用基于数据库的认证

## 使用方法

```bash
# 运行项目初始化
python init_project.py

# 初始化完成后，可以直接启动服务
python src/app.py

# 使用管理员账户登录
# 用户名: admin
# 密码: admin123
```

## 兼容性

- 保持与现有认证系统的兼容性
- 如果密钥文件不存在，会创建不带密钥的管理员用户
- 如果数据库已存在，会更新现有管理员的密钥
- 支持后续为其他用户添加密钥

## 注意事项

1. 确保数据库服务器可访问
2. 确保有足够的数据库权限创建数据库和表
3. 生成的密钥文件请妥善保管
4. 建议在生产环境中修改默认密码
